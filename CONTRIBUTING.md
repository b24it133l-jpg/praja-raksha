# Contributing to Praja Raksha

Thank you for your interest in contributing! This document provides guidelines and instructions for contributing.

## 🎯 Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Welcome diverse perspectives
- Help others learn and grow

## 🐛 Reporting Issues

**Found a bug?** Please open an [issue](https://github.com/b24it133l-jpg/praja-raksha/issues/new) with:
- Clear title describing the problem
- Detailed description of the issue
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if applicable)

## 🚀 Feature Requests

**Have a feature idea?** Open an [issue](https://github.com/b24it133l-jpg/praja-raksha/issues/new) with:
- Clear title
- Problem statement
- Proposed solution
- Alternative solutions considered

## 💻 Development Setup

1. Fork the repository
2. Clone your fork
```bash
git clone https://github.com/YOUR-USERNAME/praja-raksha.git
cd praja-raksha
```

3. Create a feature branch
```bash
git checkout -b feature/your-feature-name
```

4. Install dependencies
```bash
npm install
```

5. Make your changes
6. Test thoroughly
```bash
npm run test
```

7. Commit with clear messages
```bash
git commit -m "feat: add new feature" 
git commit -m "fix: resolve bug"
git commit -m "docs: update documentation"
```

## ✅ Pull Request Process

1. Update README.md if needed
2. Ensure tests pass: `npm run test`
3. Ensure code quality: `npm run lint`
4. Create PR with clear description
5. Link related issues in PR description
6. Wait for review and feedback

## 📋 Commit Message Guidelines

Follow conventional commits:
- `feat:` for new features
- `fix:` for bug fixes
- `docs:` for documentation
- `test:` for tests
- `refactor:` for code refactoring
- `style:` for formatting
- `chore:` for maintenance

Example:
```
feat: add citizen dashboard
```

## 🧪 Testing

All contributions must include tests:
```bash
npm run test          # Run all tests
npm run test -- --coverage  # Coverage report
```

## 📚 Documentation

- Update README.md for user-facing changes
- Add JSDoc comments for functions
- Document complex logic
- Update CHANGELOG if applicable

## 🔄 Review Process

Maintainers will:
- Review your PR within a few days
- Provide constructive feedback
- Request changes if needed
- Merge once approved

## 📞 Questions?

- Open a discussion
- Comment on related issues
- Reach out to maintainers

---

Thank you for contributing to Praja Raksha! 🎉
